# Retention template — the timing & engagement math

The data behind the rubrics. R1 (HOOK) reads first-frame rules. R2 (STRUCTURE) reads pacing math. R3 (TONE) reads pacing WPS. R5 (SFX) reads ducking & LUFS. Critique pass references retention killers.

---

## §B. Retention curve — the exact drop-off math

- **0-1.5s**: TikTok / Shorts algorithm makes its FIRST distribution decision. Fail = cold start, never escapes.
- **0-3s**: 50-60% of all drop-off happens here. 90% of underperforming reels fail in this window.
- **3-7s**: Hook resolution must be complete.
- **7-15s**: "Sweet spot" — surviving reels hold 60-80% retention.
- **12-18s**: Insert mid-video re-hook (callback, cut, surprise reveal, format shift). **MANDATORY** — re-hook absent = critique auto-fail #16.
- **15-30s**: Significant drop — retention falls to 40-60%.
- **30-60s**: Target 75-85% completion. Algorithm amplification threshold = 65-80%.

**Re-hook moves at the 12-18s mark**:
- Quick cut + camera angle change
- Background or outfit swap
- B-roll/zoom-in burst
- Verbal pattern interrupt: "But wait — this is where it gets weird"
- Visual disruption (split screen, smash cut)
- Callback teaser ("remember the number I said?")

---

## §C. Optimal length

- **Sweet spot**: 40-55s for Shorts in 2026.
- **30s @ 85% completion BEATS 60s @ 50%** definitively. Algorithm optimizes "satisfaction per swipe", not absolute watch time.
- 30s reel = ~75 words at 2.5 WPS; 45s = ~115; 55s = ~140.

---

## §D. Pacing — words per second per beat

- Industry sweet spot: 140-150 WPM = 2.3-2.5 WPS.
- Tech/AI content: 140-160 WPM (denser).
- **Hook**: 2.8-3.2 WPS (urgency)
- **Build**: 2.3-2.6 WPS (room to breathe)
- **Punchline**: 2.0 WPS or pause (deadpan demands slow)
- **CTA**: 2.2-2.5 WPS (clear, not rushed)

Critique dim 2 (pacing/effects) checks WPS by zone. Drift >15% from target triggers auto-fail #6.

## §G. CTAs that work in 2026 (ranked by signal value)

1. **DM Send / Share-with-friend** — IG #1 signal. "Send this to the friend who keeps saying AI is overhyped."
2. **Save** — IG #3 signal. "Save this — you'll need the prompt later." Educational gets 2.3× more saves than entertainment.
3. **Specific comment prompt** — beats "comment below" 3:1. "Tell me which one you'd actually use."
4. **Comment-YES (binary emotional)** — "Comment YES if this surprised you." Lower friction than keyword.
5. **Watch-Part-2** — trains algorithm to surface follow-ups.

**NEVER use** (compounding penalty across all your content for weeks): "tag 3 friends", "comment if you agree", "like for part 2", generic "comment below", "follow for more". Critique auto-fail #11.

**Best CTA for @aisimplified AI-news lane**: "Follow @aisimplified — I post the wildest AI stuff every day" (works only as news-drop register at 3-5×/day) OR "Save this for the next time someone says AI peaked." See `audience-aisimplified.md` for region-specific CTA preferences.

## §H. First-frame + cover image rules

- **Hook text on frame 0 = +50% 3-second retention.** Frame 0 must be designed (not auto-captured from b-roll). Critique auto-fail #1.
- **YouTube Shorts custom thumbnails exist (Jan 2025+) BUT do NOT show in vertical swipe feed** — frame 0 IS the de-facto feed cover.
- **IG cover gets center-cropped to 3:4 (1080×1440) on profile grid** — keep faces/text inside center 60% vertically.
- **Cover-frame match** (cover = actual opening frame) lifts save rate 25%.
- **Mr Beast face-zoom rule** holds for AI/tech: face on left third + AI demo screenshot on right two-thirds. Faces processed in ~100ms; text 60,000× longer.
- **Loop trick is mandatory now**: design frame 0 = frame N. YouTube Shorts counts each loop as a view (March 2025). Critique auto-fail #19 if violated.

## §CC. Retention quick-deploy template (the 30-55s deploy map)

| Time | Beat | Mechanic |
|---|---|---|
| 0:00-0:02 | **Hook** | First-frame headline overlay 60-80pt + pattern-interrupt SFX + caption pre-loaded. Promise payoff at 0:30. |
| 0:02-0:08 | **Setup** | Stack 2 open loops. Top-pane visual change at 0:04. Lower-third progress chip `1/3`. |
| 0:08-0:14 | **Beat 1 + first re-hook** | Deliver fact 1 + micro-payoff caption + verbal "but wait" pivot at 0:13 with smash-cut + riser. |
| 0:14-0:22 | **Beat 2 + breath** | Fact 2 + 1.5s contemplative beat at 0:20 (whisper, ducked audio). |
| 0:22-0:30 | **Beat 3 + foreshadow callback** | Fact 3 + callback to 0:00 promise + counter ticks up. |
| 0:30-0:45 | **Big payoff + emotional beat** | Reveal + avatar reaction + sub-bass impact + key-word echo + validation caption. |
| 0:45-0:55 | **Loop-back + CTA** | Visual matches frame 0. Binary CTA. Lower-third "Part 2 →" chip. |

### CC.1 Top retention mechanics by category

**Open loops (Zeigarnik):**
- "Stay until 0:30" promise hook at 0:00-0:02
- "Wait until you see #3" rank-tease (numbered counter chip 1/N)
- "But the weirdest part..." mid-roll cliffhanger at 0:18-0:22
- "And here's the punchline..." build at 0:25-0:32
- Stacked nested loops (A-open → B-open → A-close → C-open → B-close → C-close)

**Re-hook techniques (12-18s decay zone):**
- Smash-cut to new location/background (hippocampal place-cell reset)
- Sudden zoom on key word (`scale(1→1.15)` over 6f with EASE.inOut)
- Background swap mid-talking (avatar layer fixed, top-pane crossfade 200ms)
- Outfit/scene swap mid-sentence (HeyGen render variant 2)
- Format flip: talking → animation → talking
- B-roll matched to spoken word (Whisper-timestamp-driven asset switcher)

**Visual stimulation:**
- Cuts every 1.5-2.5s (P300 EEG fires per cut, +38% retention)
- Cut on every stressed word (kinetic editing — match to Whisper word timing)
- Subtle background drift (3-5px parallax per second)
- Camera shake 0.5s after major reveal (Perlin-noise wobble 15f)
- Slow Ken-Burns drift on stills (scale 1→1.05 + translate over Sequence)

**Audio attention triggers:**
- Riser 0.8-1.2s before reveal
- Sudden silence at peak (300ms before payoff word)
- Pitch drop / sub-bass on key word
- Whisper section at 0:18-0:22 (VO ducked -8dB)
- Echo/delay on key word (ffmpeg `aecho=0.8:0.88:60:0.4`)
- Reverse-cymbal 0.6s before scene change
- Sub-bass impact on every major cut (60Hz sine drop, 6f)
- Audio ducking on caption pop (sidechain VO -3dB for 8f)

**Caption retention:**
- Stacked-line buildup (each line adds a word)
- Karaoke highlight word-by-word (+23-40% watch time)
- Highlighted key word jumping (scale to 1.25× on its onset)
- Counter ticking up (numerosity-coding neurons track involuntarily)
- Stat pop-in with bounce (~2× recall vs flat text)
- Caption appears 100-200ms BEFORE audio (-6 frames from audio onset)

**Cognitive load:**
- 1-new-idea-per-5s ceiling
- Anchor + variation pattern (one motif, multiple variations)
- Triple repetition (state at 0:03, 0:20, 0:45 in different contexts)
- One visual motif throughout
- Max 2 simultaneous moving elements

**Emotional retention:**
- Awe + curiosity blend (wide-angle sweeping asset behind curiosity hook)
- Validation beat ("if you've thought about this, you're already ahead" at 0:10-0:14)
- Shared-enemy framing (each moral-emotional word = +20% diffusion)
- Mild outrage (controlled, "Why does ChatGPT still get this wrong?")
- Surprise > shock (sustainable; pre-stage boring expectation, cut to real reveal with sub-bass)
- Vulnerability beat (avatar framing tightens, music drops, single caption line)

**Algorithm-aware:**
- Loop-back trick (frame 0 = frame N) — rewatch is now TikTok's #1 ranking signal in 2026
- Sectional micro-payoffs every 7s
- Sound-off legibility audit (75-85% mobile views are silent)
- First-frame information density (60-80pt headline at frame 0)
- Series-tagged content (TikTok 2026 actively recommends "Episode 2" to E1 completers)

**Avatar-stack-specific:**
- Avatar gesture pointing at top-pane visual (deixis, +65% impact per research)
- Avatar reaction shot on punchline (expand to 60% of frame for 0.8s)
- Avatar shrink during big visual moment (1→0.7 + corner-tuck)
- Lower-third PART X / Y progress chip pinned just above avatar's hairline
- Avatar eye-contact cycling every 6s (HeyGen variants)

**Re-watch incentives:**
- Frame 0 contains a clue solved at frame N
- Multi-layer details (4-5 elements per frame at varying salience)
- Hidden text easter egg (12pt corner caption referencing channel)
- Speed-up section worth scrubbing (6-clip rapid montage at 200ms each at 0:25)

## §S. Word-level timing — 2026 best

- **WhisperX wins for word boundaries**: wraps Whisper + wav2vec2 forced alignment. Vanilla Whisper word timestamps are inference-time tricks and inaccurate.
- **Groq Whisper-large-v3-turbo**: 240× realtime, 9.7% WER. Set `timestamp_granularities=['word']`. Use for fast iteration.
- **`@remotion/install-whisper-cpp`** with `--split-on-word` and `--dtw` (whisper.cpp 1.0.55+) for measurably better timestamps. Convert audio to 16kHz WAV first.
- **`createTikTokStyleCaptions({captions, combineTokensWithinMilliseconds: 1200})`**: chunks word array into "pages." Lower ms = 1-3 word bursts (viral feel); higher = 6-8 word lines. Each token's `text` MUST include leading space; render with `white-space: pre`.

## Retention killers (auto-fail or near-auto-fail)

- Silent first 0.5s (~30-40% lost immediately)
- Logo intro / "in this video..." pre-roll (-30-40%)
- Repeating title verbally at start (redundant load)
- Talking-head only 5+ seconds (max 3s pure avatar without top-pane variation)
- Cut to b-roll without sound design
- SFX mismatching motion direction (whoosh L→R must match motion L→R)
- Text appearing AFTER spoken word (must lead by 100-200ms)
- Slow build-ups / fades (algorithmic kiss of death)
- Vague hooks ("this changed everything")
- Mismatched promise vs payoff (algorithm trust erosion)

These are the "do NOT" companions to the §J/§K/§L creative catalogs. Check the script and beat plan against these before render.
