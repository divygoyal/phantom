# Script-writing — how to write reel scripts that go viral

This file is the **drafting playbook** for `script.txt`. Load it in Phase 2.1 before writing the first beat. The rubrics tell you tone and structure; this file tells you how the *words* on the page must sound when an avatar says them.

A reel script is not an article. It is not a product description. It is not a teleprompter for a corporate explainer video. It is a friend's voice note, transcribed.

---

## §0. The four locked rules — never violate

These four rules are non-negotiable. Phase 3 critique auto-fails any draft that misses one.

1. **Read every line aloud.** If any line sounds like *narration* — like an article being read — rewrite it. The test: would a friend actually say this sentence on a voice note? If no, kill it.
2. **First 5 words are the catch.** Word 1-5 must stop the scroll on their own. After "humanoid robots have hobbies now" the viewer has already locked in. After "i'm just gonna casually mention" the viewer has scrolled.
3. **Every beat is one thought.** No subordinate clauses. No "X, which is Y." Telegraph cadence. One clause per fragment, max two.
4. **The ending is a callback + reframe.** Loop the visual back to the hook. Reframe the topic-meta in one line. Future-projection or topic-rebuttal lands hardest.

---

## §1. The first 5 words — the catch

The catch is the first 5 spoken words after the hook visual locks. The avatar's first 1.5 seconds.

**The catch must do three things at once:**
1. Name the subject (specific noun — not "AI", but "Unitree" / "Codex" / "Nano Banana").
2. Punch with an unexpected verb (not "introduces" — try "fishes", "drops", "ate", "killed", "filmed").
3. Lock-in the visual frame the cold-open delivered.

**Patterns that work:**

| Pattern | Example | Why |
|---|---|---|
| `[Subject] just [verb-ed] [object].` | "Unitree's robot just learned to fish." | Names + verbed + object — three concrete nouns |
| `[N] of [thing] just [event].` | "Five years of progress just collapsed." | Specific number + collapse-verb |
| `[Class] have [unexpected verb] now.` | "Humanoid robots have hobbies now." | Class-statement + impossible verb |
| `Bro. [Specific claim].` | "Bro. A Chinese robot just learned to fish." | "Bro" direct address + claim — Indian-English-millennial cadence |
| `[Action verb]. [Specific subject]. [Result].` | "Watch this. Codex shipped pets." | Telegraph-cadence triple |

**Patterns that DON'T work** (these are catches with no catch):

- "I can't believe i'm saying this..." — meta-disbelief without naming the subject
- "Okay so basically..." — "basically" is the AI-narrator tell
- "Today I want to talk about..." — anti-opener
- "This is insane..." — generic, no specific noun
- "If you're an AI builder..." — persona-segmenting opener (sounds like an ad)
- "Most people don't know..." — insider-claim without specificity

**Read aloud test:** if the first 5 words of your draft hook are vague enough that they could open a hook on a *different topic*, rewrite. The catch must be UNIQUE to this story.

## §2. The conversational register — friend talking, not narrator

A viral reel script reads as if transcribed from someone's voice note to a friend. Specific markers of the friend-talking voice:

| Friend-talking | Narrator (reject) |
|---|---|
| "okay so a year ago this thing was learning ballet" | "Twelve months ago this same robot was in ballet class." |
| "march, milan's cathedral, michael jackson moves" | "By March it was at Milan's Duomo. Doing Michael Jackson." |
| "and it's sixteen grand. you can literally just buy one." | "And the kicker? It's sixteen grand. You can order one tomorrow." |
| "they keep dropping a new skill like every two months" | "Unitree drops a new humanoid skill every two months." |
| "and then this guy goes and builds a whole thing" | "A creator built an entire ecosystem." |
| "wait. wait. it does what?" | "Surprisingly, the robot can do X." |

**Friend-talking moves to deploy** (each is a tell of a real human voice):

- **"like" as connective tissue** — "like a year ago", "it's like, fishing now", "that's like, the kicker"
- **filler words that earn their place** — "okay so", "look", "wait", "honestly", "no but", "i mean"
- **trailing pauses** — "and now... it's fishing." The pause is the joke.
- **rising intonation as pacing** — "april? rocket skates. in hangzhou."
- **fragment cadence** — "march. milan's cathedral. michael jackson." Period-as-pause beats every 2-4 words.
- **direct viewer address** — "you can literally just buy one", "show them this"
- **mid-thought asides** — "(which is wild because—)", "and i'm telling you—"
- **self-correction** — "ok wait so the actual—"
- **peer addressing** — "guys", "bro", "y'all" (use sparingly; matches Varun-Mayya / Indian-English-millennial register)

**Narrator-cadence tells to KILL** when you spot them in your own draft:

- Articles where contractions would land ("a year ago" → "year ago" / "last year")
- Date-as-opener ("Twelve months ago, a Chinese company..." → "okay so a year ago...")
- Subordinate clauses ("...which is when they put it on rocket skates" → "...april. rocket skates.")
- "I'm telling you guys" tics that lock per-beat (Varun-leaning is fine, but don't stack — once per reel max)
- Triple-clause beats ("It dances, it skates, it fishes" → fragment them across separate beats)
- Product-page phrasing ("Ships from Hangzhou. Tax not included." reads as marketing — only works if the deadpan is bone dry)
- "I want to talk about" / "let me tell you" preambles
- "Here's the thing" pivot phrase (used to be powerful; now reads exhausted-coded)

## §3. The Varun-Mayya register — when to crank confidence

The default register is *casual conversational*. The Varun-Mayya register is a high-confidence-conversational variant useful for:

- Multi-beat showcase reels with named entities
- Hot-take reels where the channel is asserting a position
- Industry-disruption reels where stakes need to be communicated quickly
- AI-news reels for AI-builder audience persona (the cohort responds to confident posture)

**The Varun-Mayya signature moves:**

| Move | Example | Where it lands |
|---|---|---|
| `Bro. [Claim].` opener | "Bro. A Chinese robot just learned to fish." | Reel start, direct address |
| `Look at this.` / `Look at the timeline.` | "And this is not a fluke. Look at the timeline." | After the catch, before the build |
| Indian-English construction | "A year back" (vs "a year ago") | Date-references; localizes the voice |
| `Casually [verb-ing] [place]` | "Casually rolling around Hangzhou." | Geographic punch; deadpan tag |
| `Zero X. Zero Y.` repetition | "Zero bathroom breaks. Zero phone." | Triple-beat zinger; emphasis pattern |
| `[Subject] drops [thing] every [interval].` | "Unitree drops a new humanoid skill every two months." | Frame-the-curve beat |
| `And the kicker?` pivot | "And the kicker? It's sixteen grand." | Mid-reel turn into the stake |
| `Give it [N] more months. [Prediction].` | "Give it twelve more months. It's fishing with you." | Prediction-close ending — Varun signature |
| `[Topic-meta] isn't [common claim]. It's [reframe].` | "AI didn't peak. It just learned to fish." | Reframe sweep — alt prediction-close pattern |

**Varun-Mayya pacing:** fast confident assertion, no hedge phrases. Avoid "I think", "maybe", "kind of", "sort of". The voice declares.

**Varun pitfalls to avoid** (so we don't drift into course-bro):
- Don't stack "I'm telling you" / "let me tell you" / "here's the thing" — pick one per reel max.
- Don't use "incredible / insane / mind-blowing" — banned per audience taboos.
- Don't overuse "Bro" / "Guys" — once at the open is fine; sprinkling them in middle beats reads try-hard.
- Don't end every line on a rhetorical question — the prediction-close needs a flat-declarative landing, not "...what's next?"

## §4. The ending — callback + reframe + WOW

A great close does three things in one breath:
1. **Loops the visual back to the hook.** The last word echoes the first word, OR the visual repeats the cold-open clip.
2. **Reframes the dominant topic-meta.** The viewer entered the reel with prior beliefs; the close updates one.
3. **Projects forward** OR **rebuts a position** OR **makes the viewer the subject** — pick one.

**Three close patterns that work:**

### Pattern A: Prediction-close

`Give it [N] more months. It's [outcome] with you.`

- Mirrors the timeline setup. "A year back" → "Give it twelve more months."
- "With you" makes the viewer the subject.
- The outcome word is the visual callback.

Example: "Give it twelve more months. It's fishing with you." — visual returns to the fishing clip; the viewer is now part of the scene.

### Pattern B: Topic-meta rebuttal

`[Common claim] isn't [verb]. It's [reframe with hook callback].`

- The common claim is the discourse the viewer entered the reel with ("AI peaked", "AI is plateauing", "AGI is hype").
- The reframe inverts it.
- The reframe word is the visual callback.

Example: "AI didn't peak. It just learned to fish." — rebuts "AI peaked" + visual loops to fishing.

### Pattern C: Action-prompt with implicit save

`Next time someone says [X] — show them this.`

- Concrete share-trigger.
- "Show them this" implicitly = save the reel for that conversation.
- Avoids the dead "follow for more" / "comment AI" CTAs.

Example: "Next time someone says AI peaked — show them a robot fishing." — concrete + share-coded.

**Closes that DON'T work:**
- "What do you think?" — open question with no anchor; viewer scrolls.
- "Comment your thoughts below!" — course-bro tail.
- "Follow for more!" — anti-CTA; loses save-CTA priority.
- A new fact in the close — adds cognitive load right when viewer should be feeling something.
- A hedge — "anyway, that's all..." kills the punch.

**The WOW threshold:** a great close gets the viewer to either pause-the-scroll, send the reel, or comment with their own riff. If the close just *describes* what happened — "and that's how Unitree taught a robot to fish" — it's a wrap-up, not a WOW. Rewrite.

## §5. The 6-pass drafting flow

Walk these passes in order. Skipping any pass leaves narrator-cadence in the script.

### Pass 1 — Skeleton (the fact list)

Write the script as a bare fact list, one fact per beat. No tone, no voice. Just: B1 = X, B2 = Y, B3 = Z. This is the *information layer*. If you can't list facts cleanly, the topic is too thin — return to Phase 0.

### Pass 2 — Catch (the first 5 words)

Drop everything else. Spend 5 minutes only on B1's first 5 words. Write 8 candidate catches. Read each aloud. Keep the one that stops your own scroll.

### Pass 3 — Voice (rewrite as friend talking)

Take the skeleton from Pass 1 and rewrite each beat as a friend's voice note. Use §2 friend-talking moves. Drop subordinate clauses. Add filler that earns its place. Read aloud after each beat.

### Pass 4 — Punches (highlight specific moments)

Mark the comedy/punch beats. Each gets one sharp word: a place name, an absurd verb, a deadpan tag. Examples: "Casually." / "Michael Jackson moves." / "Tax not included." / "for fun." Don't punch every beat — pick 2-3.

### Pass 5 — Ending (callback + reframe)

Write 3 candidate closes using the Pattern A/B/C frameworks above. Pick the one that loops the visual + reframes the meta. Test: would a viewer screenshot the close-line as a quote? If no, rewrite.

### Pass 6 — Read-aloud audit

Read the entire script in one breath. Mark every line that sounds like narration. For each marked line, ask: would a friend actually say this on a voice note? If no, rewrite.

If you finish the read-aloud and feel like the script flows as one continuous voice-note, ship to Phase 3 critique. If any line still reads written, return to Pass 3.

## §6. The delivery-style script

After the read-aloud audit passes, also produce a **delivery-style script** for Stage 2 recording. Format:

```
B1  [voice tag, mood]
> The line, with **CAPS** for vocal emphasis and *—* for micro-pauses and *...* for trailing pauses.
> Delivery note: where to hit, pace, tone, beat-behavior.
```

Per-line conventions:
- **CAPS** = vocal emphasis (the avatar/recorder hits this word harder)
- **—** = micro-pause (~0.3s)
- **...** = trailing pause (~0.6s)
- **[beat]** = full pause (~0.8s) — used between fragments and before punches
- EL voice tag at the head — drives mood. Single tags preferred; max 2 stacked.

The delivery-style script is what the user records against. Each beat should fit a single breath. Delivery notes tell the recorder where to hit, when to pause, what tone.

Also produce a clean **inline-tagged version** suitable for HeyGen TTS:

```
[urgent] Bro. [pause] A Chinese robot just learned to fish.
[confident] And this is not a fluke — look at the timeline.
...
```

That goes directly into HeyGen's input field if the user is generating TTS.

## §7. Common drafting bugs (and the rewrites)

These are the failure modes I keep hitting. Each has a stock rewrite move.

| Bug | Symptom | Rewrite |
|---|---|---|
| **Date-as-opener** | "Twelve months ago this same robot was in ballet class." | "okay so a year ago this thing was learning ballet." |
| **Article overload** | "And this is not a fluke. Look at the timeline." | "and look — this isn't a one-off. look at the timeline." |
| **Triple-clause beat** | "It dances, skates, and fishes." | Fragment it: "march? dancing. april? skates. now? fishing." |
| **Product-page tag** | "Sixteen grand. Ships from Hangzhou. Tax not included." | "and it's sixteen grand. you can literally just buy one." |
| **Course-bro hype word** | "This is INSANE." | Drop entirely; let the visual carry. Or: "this is wild." |
| **Generic CTA** | "Save this and follow for more!" | "next time someone says X — show them this." |
| **Question-mark close** | "What do you think? Are robots taking over?" | Declarative prediction-close. |
| **Symmetric beats** | every beat is 12 words | Vary length: 4w / 11w / 6w / 4w / 9w — front-load weight, breathe between |
| **Three filler tags in a row** | "i'm telling you / look at this / honestly bro" | Pick ONE filler per reel as the recurring voice-tic; cut the others |
| **First-5-words = stock phrase** | "i can't believe i'm saying this" / "this is wild" | Rewrite with named subject + unexpected verb in the first 5 words |
| **No catch-back ending** | close adds new fact | Rewrite as Pattern A/B/C close |

## §8. The script-writing checklist (to run before Phase 3 critique)

- [ ] First 5 words name a specific subject + an unexpected verb.
- [ ] Every beat is one thought, no subordinate clauses.
- [ ] Read-aloud passes: every line sounds like a friend talking.
- [ ] No "incredible / insane / mind-blowing" anywhere.
- [ ] No anti-openers (Hi/Hey/So/Today/Welcome/Let me).
- [ ] Filler ("like / okay so / honestly / wait") is present where it earns its place — not stacked.
- [ ] At least one mid-reel comedic punch (place-name / absurd verb / deadpan tag).
- [ ] Re-hook beat at 12-18s mark (a "but wait" / format-flip / callback).
- [ ] Closing line is Pattern A (prediction), B (topic-meta rebuttal), or C (action-prompt).
- [ ] Closing word echoes the hook word (visual callback).
- [ ] Last word is hard-stop, not a question or hedge.
- [ ] Delivery-style script produced with CAPS / em-dashes / [beat] markers and EL tags.
- [ ] Inline-tagged version produced for HeyGen TTS input.

If any item fails, rewrite. Cost of rewriting before Phase 3 is 5 minutes; cost of rewriting after a failed render is 30 minutes.

## §9. Tutorial / recipe / workflow content — the one-continuous-voice pattern

(Validated 2026-05 against the higgsfield-2052062257 reel — Higgsfield MCP agentic-ads workflow with 6 sequential tool surfaces.)

Tutorial-howto content has a unique failure mode the prior 8 sections don't catch: writers default to **stepwise enumeration** ("Step one — X. Step two — Y. Step three — Z."), which reads as narrator-cadence and fails the §0 read-aloud test even when each individual line otherwise looks clean. The cadence is the problem, not the content.

**The rule for tutorial/recipe/workflow scripts: write the entire body as ONE continuous thought.** No numbered steps. No section headings. The viewer follows the actions naturally as the script flows; arrival at the conclusion happens *in the viewer's head*, not at a labeled landing.

### What stepwise enumeration looks like (REJECT)

```
[matter-of-fact] Step one — Claude Cowork digs through Meta's Ad Library. Learns what's working.
[curious] Then Higgsfield MCP cooks the creatives. Video, image, whatever.
[matter-of-fact] Visuals drop straight into Meta Ads Manager. No designer in the loop.
[confident] Meta MCP launches the campaign. Watches the numbers roll in.
[deadpan] Winners scale automatically. Losers die quiet. Higgsfield handles both.
[impressed] And the whole thing — runs every Monday. While you sleep.
```

Why this fails the read-aloud:
- Every beat opens with a stage marker ("Step one —", "Then", "Visuals drop straight", "Meta MCP launches", "Winners scale automatically", "And the whole thing"). Each line introduces itself as a new step.
- The avatar is performing a *list*, not telling a *story*. There is no narrative momentum carrying the viewer from line 2 to line 3 — only stage-marker conventions.
- The realization beat ("runs every Monday") arrives as another item, not as the emotional thesis the viewer earned.
- User feedback when this shipped to draft: *"the mid script is not human-like, whole script should be one flow and user gets the point of the video and comes up with conclusion."*

### What one-flow-voice looks like (SHIP)

```
[impressed] Claude is running my paid ads now.
[matter-of-fact] Higgsfield dropped this thing called MCP last week. Gives your agent every tool, every API.
[matter-of-fact] So Claude opens Meta's Ad Library, pulls apart what competitors are doing.
[curious] Generates the creatives. Pushes them straight to Ads Manager.
[confident] Meta MCP launches the campaign, watches the numbers.
[deadpan] Winners scale. Losers die quiet.
[impressed] And here's the part that broke my brain — it runs every Monday. While I'm asleep.
[dry] So yeah. Give it six months — your media buyer is Claude too.
[enthusiastic] Save this. Drop HIGGSFIELD for the setup.
```

Why this works:
- **No stage markers.** Connective tissue ("So", elided "Then" via period-as-pause, "And here's the part") flows between actions without announcing the next step.
- **Sentence fragments do the punching.** "Generates the creatives. Pushes them straight to Ads Manager." — the implicit subject ("Claude") is reused without restatement. Viewer fills in the gap; the demo carries the visual.
- **The realization beat uses a friend-aside, not a section break.** "And here's the part that broke my brain —" is the pre-warn cue that tells the viewer *the next line is the load-bearing one*. They lean in. Then "runs every Monday. While I'm asleep" lands — and the viewer reaches the "agents run my whole pipeline autonomously" conclusion *before* the dark-humor close labels it.
- **The close labels the conclusion the viewer just had.** "So yeah. Give it six months — your media buyer is Claude too." — speaker is now articulating what the viewer already thought. Audience feels smart, not lectured.

### The three validated phrase patterns from this session

#### A. The hook line — `[Subject] is [verb-ing] my [object] now.`

Validated example: **"Claude is running my paid ads now."**

Why it works:
- Names the tool in the first word (SEO + cohort routing for AI-builder audience).
- Specific verb ("running" not "doing"). "Running" is operationally concrete.
- "my" — personal stake. Not "your" / "the" / "an ad". Speaker has skin in the game; viewer aligns.
- "now" — timestamped urgency. Adds: this is happening, the topic is current.
- 7 words. Under §1's first-5-words rule (word 1 names subject, word 3 is the unexpected verb).

Anti-patterns this beats:
- `[Tool] just dropped X.` — launch-cadence; wrong for a "this is happening to me" hook.
- `Look at this — [Tool] does X.` — show-and-tell cadence; weaker stake.
- `[Tool] just changed [industry].` — abstract claim; viewer skips.

#### B. The realization-pivot — `And here's the part that broke my brain — [reveal].`

Validated example: **"And here's the part that broke my brain — it runs every Monday. While I'm asleep."**

Why it works:
- Pre-warns the viewer that the next clause is load-bearing. Buys 0.5s of leaning-in attention before the reveal.
- "broke my brain" is friend-emotional, not narrator. A real human said this.
- The em-dash is the gear shift — followed by the actual reveal.
- "While I'm asleep" is the dark-humor undercut already half-baked in; the dry close that follows just labels it.

Use this exactly once per reel, at the beat where `emotional_thesis` lands. Stacking it twice burns the move.

Alternative friend-aside patterns (rotate to avoid wear-out):
- "And the actually wild part — [reveal]."
- "And this is the bit that got me — [reveal]."
- "Wait. Wait. — [reveal]." (Indian-English / Varun-Mayya register)

#### C. The prediction close — `Give it [N] months. Your [role/object] is [subject] too.`

Validated example: **"Give it six months — your media buyer is Claude too."**

Why it works (Pattern A from §4):
- Mirrors a timeline ("dropped last week" → "give it six months"). Time as narrative spine.
- "Your" makes the viewer the subject. They're now inside the story, not watching from outside.
- The callback word is the *hook noun*. Hook: "Claude is running my paid ads." Close: "your media buyer is **Claude** too." Loop closed.
- "too" implicates the viewer in the same fate as the speaker. Forecloses argument ("you'll feel this the same way I do").
- 11 words. Pre-tested for read-aloud cadence.

Anti-patterns this beats:
- "And that's how agents run paid ads in 2026." — wrap-up cadence, no WOW.
- "What do you think?" — open-question close; viewer scrolls past.
- "Save this if you want to learn more." — pre-CTA hedge; the CTA itself should be the close.

### Save-CTA convention for tutorial reels

Tutorial content is **reference content**. Viewers expect to rewind / save / share with the friend who'll need this next month. CTA must be save-coded:

```
[enthusiastic] Save this. Drop [KEYWORD] for the setup.
```

- "Save this." — direct save instruction, 2 words.
- "Drop [KEYWORD] for the setup." — keyword funnel + concrete deliverable ("the setup" not vague "the link").

NOT acceptable for tutorial reels:
- "Follow for daily AI" (course-bro; save-CTA priority always wins for reference content per `references/audience-aisimplified.md`).
- "Hit the bell" (declining pattern).
- "What do you want me to cover next?" (open-question; not actionable).

### One more thing — render-side requirements that protect the script

A script can flow perfectly and still get sabotaged by the asset framing:

1. **Demo footage readability** — If the body shows real screen-rec dashboards (text labels, numbers, prompt windows), the viewer will rewind to read them. Use `assetMode: 'card'` (letterbox) for 16:9 landscape sources. `assetMode: 'fullbleed'` crops the sides and slices off dashboard text. Validated 2026-05 — see `references/render-cookbook.md` §Z.
2. **The hook stays fullbleed** — Hook B1 + close B8 (Dario callback) use portrait-coded composition where the subject is centered. Fullbleed cover-fit works. Only body beats with horizontal landscape source need card mode.

### Add to §8 checklist (Phase 3 critique reads this)

After the existing checklist items, add for tutorial-classified reels:

- [ ] **No stepwise enumeration** — no beat opens with "Step one /  Step two / Then we / Next we / The first thing".
- [ ] **One continuous voice through body beats** — beats flow as fragments and connective tissue, not as labeled sections.
- [ ] **Realization-pivot present at `emotional_thesis` beat** — uses friend-aside cue, not section break.
- [ ] **Close uses Pattern A prediction format** (validated for tutorial-howto specifically).
- [ ] **CTA is save-coded + keyword funnel** — not follow-ask.
- [ ] **Body asset mode is `card`** (letterbox) if any landscape demo source has readable text/dashboards.

## §10. Register-failure modes (validated 2026-05-12 from the veo4vs-2053857256 session)

This section catalogs the **three failure modes** the script drafting loop kept slipping into, the **diagnostic signal** for each, and the **working patterns** that displaced them. Every reel goes through these — when a draft "doesn't sound human," it's one of three.

The register hierarchy (escalating from broken to landing):

```
narrator      ← stilted, "today we examine"
   ↓
founder-explainer ← TED-talky, parallel aphorisms
   ↓
friend talking  ← confident declarative, no filler stacking
```

Most drafts crash into the middle layer (founder-explainer) and feel ALMOST right but obviously written. The fix is dropping into friend-register without overcorrecting into stacked-filler caricature.

---

### §10.1 Failure mode A — written-aphorism close

**Signal**: the close is a two-clause parallel contrast that reads as a quotable line, not as something a person says out loud.

**Rejected (this session)**:
- `Voice is solved. Teaching isn't.`
- `The model's free. The lesson plan isn't.`
- `Models become cheap. Lesson plans don't.`

Why they fail: they're *aphorisms* — written rhetorical forms optimized for screenshot, not voice-note. Real friends don't speak in maxims; they ramble toward conclusions.

**Working (this session)**:

> `So no, teachers didn't become optional. Bad educational content did.`

Why this lands instead:
- Connects to the hook claim via `So no` — natural spoken pivot.
- Specifies the *real* claim (the narrowing) — viewer hears "ah, that's what they meant."
- Pattern B topic-meta rebuttal (see `script-writing-saas-angle.md` §10) — inverts the hook's overstated claim.

**The fix when you spot this in a draft**: replace the parallel-aphorism with a **callback that narrows or inverts the hook**. The hook overclaimed slightly; the close says the real version.

---

### §10.2 Failure mode B — formula-transition phrases

**Signal**: every pivot beat opens with a stock phrase — "Catch is —", "Here's the thing —", "Here's the part that gets me —", "The thing is —".

**Rejected (this session)**:
- `Catch is — it can deliver a voice. It can't write the lesson.`
- `And here's the part that gets me — neither one can write a lesson.`

Why they fail: these are SaaS-explainer formulas. Used once they're invisible; stacked across beats they read template-y. Even the canonical Varun move ("And here's the part that broke my brain —") wears out fast.

**Working (this session)**:

> `But here's the twist. Neither of these tools is actually teaching. They're performing.`

Why this lands:
- `But here's the twist.` is a setup-payoff form — buys 0.5s of leaning-in BEFORE the reveal lands.
- Pre-warns "the next sentence is the load-bearing one" without sounding formulaic.
- Then telegraph-fragment reveal: `They're performing.` (two words, single-verb reveal).

**The fix when you spot this**: use `But [single connector word]` + a clean declarative sentence. Or just drop the transition phrase entirely and let the sentence start directly with the new claim.

---

### §10.3 Failure mode C — stacked filler tics

**Signal**: a single beat (or worse, the whole reel) opens every line with a different filler — "Bro. Look —. Like, honestly. Yeah no. So basically."

**Rejected (this session)**:

```text
Bro. Google just dropped Veo 4. Look — it just made teachers optional.
Watch — same prompt, two AI models. Veo 4 first. Same professor, two cuts. The chalk doesn't change. The voice doesn't shift. Listen to it. This used to be impossible.
```

Three different attention-grabs (`Bro` + `Look —` + `Watch —` + `Listen to it`) in the first two beats. The script reads as a caricature of friend-talking rather than friend-talking.

Why it fails: real speech has **one** recurring tic per voice-note, not five. Stacking them signals "I'm trying to sound casual."

**Working (this session)**:

> `Google just dropped Veo 4, and it might have just killed boring educational videos.`
> `Same prompt. Two generations. Same teacher. Same voice. Same handwriting. Same classroom.`

Zero filler tics. Just confident declarative + anaphoric rhythm (`Same X. Same Y. Same Z.`). The friend voice comes from **rhythm and confidence**, not from cosplaying with `Bro / Look / Like`.

**The fix when you spot this**: rewrite the offending beats stripping all filler. Add ONE filler back ONLY if a specific moment needs it (a reaction beat, a punch beat). Filler is seasoning, not the main dish.

---

### §10.4 Working patterns — what actually lands in friend register

The five patterns that displaced the failure modes during this session:

#### Pattern 1 — Two-word hedge

`Now Veo gets dangerously close from one sentence.`

`Dangerously close` is the move. Confident (acknowledges Veo is impressive) but humble (didn't overclaim "Veo nailed it"). Stops the "AI-bro hype" tell without losing the awe.

Other two-word hedges that work: `oddly believable`, `quietly devastating`, `almost indistinguishable`, `surprisingly competent`, `genuinely terrifying`.

#### Pattern 2 — Anaphora repetition for rhythm

`Same teacher. Same voice. Same handwriting. Same classroom.`

Four parallel `Same X` fragments build hypnotic rhythm that lands the technical claim ("Veo holds character across cuts") through pure cadence — viewer isn't reading the words, they're feeling the repetition.

Other anaphora skeletons: `Same X` × 3-4, `No X` × 3, `Just X` × 3, `It's [emotion] / It's [emotion] / It's [emotion]`.

#### Pattern 3 — Visceral sensory verb

`You can feel the illusion falling apart.`

`Feel the illusion falling apart` beats `you can hear the voice slip` because:
- Multi-sense ("feel" not just "hear")
- Specific noun phrase ("the illusion") that names the abstract thing
- Visceral verb ("falling apart") with kinetic energy

Other visceral verb patterns: `the seams show`, `the spell breaks`, `the magic curdles`, `the illusion collapses`.

#### Pattern 4 — Single-word reveal after setup

`But here's the twist. Neither of these tools is actually teaching. They're performing.`

The reveal is **two words** (`They're performing.`). Not a metaphor ("an actor with a chalkboard"). Not an explanation. Just the verb.

Single-word reveals work because the setup did all the work. The reveal punches without bloat.

#### Pattern 5 — 4-item concrete enumeration

`The hard part is still the lesson: what to explain, what example to use, where students get confused, and how to make the concept click.`

Four concrete items naming what humans do that AI can't. NOT "curriculum" or "pedagogy" (abstract jargon). NOT three items (too short). NOT five (too listy). Four items with the final item being the strongest (`make the concept click`).

This is the move when you need to name what stays valuable in a category being disrupted. Don't say "the lesson plan still matters" — say what the lesson plan IS.

### §10.5 The flow check

After drafting, read the whole script aloud as one continuous voice note. Then ask:

> **Does each sentence earn the next one?**

If the answer is "yes, because X implies Y" — the sentence is earned.
If the answer is "I wrote it next because I'm explaining Z" — the flow is broken.

Concrete check using the veo4vs reel:

| L | Earns L+1 how? |
|---|---|
| L1 "killed boring educational videos" | Sets up the audacious claim that needs evidence |
| L2 "Same prompt. Two generations. Same teacher..." | Shows the technical evidence the claim depends on |
| L3 "Here's the same prompt on Seedance" | Pivots from Veo to the competitor for the comparison |
| L4 "Not bad. But the voice slips..." | Friend's actual reaction-after-watching |
| L5 "But here's the twist. Neither is actually teaching." | Pivots from MODEL-vs-MODEL to MODEL-vs-HUMAN — surprise re-frame |
| L6 "So no, teachers didn't become optional. Bad educational content did." | Resolves the L1 overclaim with the actual claim |

Every sentence is the natural next thing a friend would say. That's the flow.

---

### §10.6 Hook-narrowing for Pattern B callback close

When the close uses Pattern B (topic-meta rebuttal), the hook needs to **slightly overstate** the claim so the close has something to correct.

**Don't write the precise claim at the hook**:
- `Google just dropped Veo 4. The lesson plan is the new moat.` ← precise, no room for callback

**Do overstate the hook** with a hedge word (`might`, `looks like`, `feels like`):
- `Google just dropped Veo 4, and it might have just killed boring educational videos.` ← overstated, with hedge, sets up the callback

**Then the close inverts**:
- `So no, teachers didn't become optional. Bad educational content did.`

The viewer's brain does the work: they noticed the hook's overclaim (because the hedge signaled "I'm not 100% on this"), and the close gives them the precise version. The arc IS the argument.

This is why hooks should be slightly louder than the close, not the other way around.

### §10.7 Add to §8 checklist for hot-take / comparison reels

- [ ] **No written-aphorism close** — final beat is not a screenshot-quotable parallel contrast. Use Pattern B callback or Pattern A prediction instead.
- [ ] **No stacked filler tics** — at most ONE recurring filler/tic per reel (`bro` OR `look` OR `like`, never all three).
- [ ] **No formula-transition phrases** — pivots use single connector words (`But`, `So`, `Now`, `Yeah`) instead of `Catch is —` / `Here's the thing —` / `Here's the part that gets me —` patterns.
- [ ] **Credible caveat present** — script acknowledges what the new thing CAN'T do (see `script-writing-saas-angle.md` §5).
- [ ] **Hook overclaims with hedge** if close uses Pattern B callback — the hook should have an explicit hedge (`might`, `looks like`) and the close should narrow.
- [ ] **At least one two-word hedge in the body** — `dangerously close`, `oddly believable`, `quietly devastating` etc. Single hedge per reel.
- [ ] **Anaphoric repetition in evidence section** — when listing observable claims (voice, handwriting, character consistency), use parallel `Same X` or `No X` fragments for rhythm.
- [ ] **Reveal is single-word or short fragment** — the caveat/twist payoff after the setup uses 1-3 words max (`They're performing.` not `They're putting on a performance with a chalkboard backdrop.`).
- [ ] **Read-aloud flow check passes** — every sentence earns the next; if any beat doesn't follow from the prior, rewrite.

## §11. The three script-density registers — picking the right one (validated 2026-05-14 from the hfsuper-2054808204 session)

Through three production cycles we've identified three distinct script-density registers. Each suits a different reel type. Picking the wrong register for the topic produces a reel that's either too dense (loses sound-off viewers) or too sparse (reads as ad copy).

### §11.1 The three registers — side-by-side

Same demo beat ("scans competitor ads on Meta and YouTube and ranks them") in each register:

| Register | Words/beat | Beat duration | Example |
|---|---|---|---|
| **Punchy** | 8-12 | 3-4s | `"Scans Meta Ads and YouTube. Finds what's already winning."` |
| **Middle-ground** | 12-18 | 5-7s | `"Then it scans your competitors' ads on Meta and YouTube — every winning creative from the last thirty days."` |
| **Explanatory** | 20-30 | 7-10s | `"Then it scans every ad your competitors are running on Meta and YouTube — anything that's been live for thirty days, because that's how you tell what's actually working."` |

### §11.2 What each register optimizes for

**Punchy** — feature *naming* register. Best when:
- Topic is single-take or hot-take (`hot-take-opinion`, `drama/discourse`, `culture-meme`)
- Visual is self-explanatory (the hero clip carries the meaning)
- Channel posture is sceptical-fast (Petdex deadpan, Veo4vs comparison)
- Viewer already knows the brand/category

**Middle-ground (DEFAULT for feature-drop / model-launch / research-paper)** — feature *demonstration* register. Best when:
- Topic is multi-demo source video (Higgsfield Supercomputer, Googlebook, DeepMind cursor)
- Each beat shows a complete demo that needs framing
- Sound-off viewer needs the avatar voice to orient them ("this is competitor scanning")
- Tonal palette is warm-humor or sincere-awe (not deadpan)

**Explanatory** — feature *mechanism* register. Best when:
- Topic is tutorial-howto OR research-paper deep-dive
- Audience is AI-builder lean (wants to know HOW, not just WHAT)
- Beat duration ≥8s (per the Phase 0.5 source catalog)
- Visual is complex UI requiring narration

### §11.3 Middle-ground register rules (the new default)

- **12-18 words per beat.** 5-7s beats. Avatar pacing ~2.0-2.4 WPS at HeyGen's actual delivery.
- **One light connector per beat.** Approved set: `Then`, `Then it`, `So`, `And`, `Now`, `The way it works`, `That's how`, `So yeah`, `Here's the thing`. Use ONE per beat, not stacked.
- **Specifics retained.** Numbers (`40+`, `three layers`), names (`Meta Ads`, `YouTube`, `Claude`), filters (`thirty days`, `every Monday`), proper nouns (`Hermes Agent`, `Gemini`).
- **Mechanism explanation skipped.** Avoid `because X means Y` and `which means` — those push the beat into explanatory territory. The viewer learns the WHAT, not the WHY.
- **Continuity glue between beats.** The first 1-3 words of each beat should hint at the previous beat's outcome. Example chain: `"Then it scans..."` → `"Every Monday, three fresh ad ideas..."` → `"Forty-plus tools..."`. Each beat's opener earns the previous beat's payoff.

### §11.4 Worked example — Higgsfield Supercomputer reel (validated v3, 14 beats, 50.85s)

Demonstrates middle-ground register with hook-first opener + AI-creative-studio close:

```
[hook, shocked] Higgsfield is not an AI video tool anymore.
[hook, payoff] It's an ad agency. Compressed into one AI.
[lean-in] You give it a product. Watch.
[building] Reads your landing page. Pricing, tone, audience.
[building, faster] Scans Meta Ads and YouTube. Finds what's already winning.
[impressed] Every Monday, three fresh ad ideas. Before you wake up.
[specific] Forty-plus tools. Three memory layers. One agent.
[demo] Need a podcast? Writes it. Voices it. Plugs in your product.
[demo] Need ad variations? Generates them.
[demo] Need a content calendar? Schedules it.
[whisper] Remembers your brand. Every output gets sharper.
[source] Here's the official launch tweet. Available now.
[final punch] Higgsfield didn't ship a feature. They shipped a creative studio.
[CTA] Save this. Full breakdown at aisimplified.
```

Most beats are 8-13 words (middle-ground low-end). The `[demo]` triplet uses parallel `Need X? Verb it.` structure for rhythm. The close lands the reframe (`creative studio`) that echoes the hook (`ad agency`) — Pattern B topic-meta rebuttal.

### §11.5 Register failure modes (when the wrong register breaks the reel)

- **Punchy on a source-first feature-drop:** avatar names features faster than UI demos resolve. Sound-off viewer sees a UI flash + 3 caption words and doesn't process either. Loses retention at the demo beats.
- **Middle-ground on a hot-take:** the connector words dilute the take's punch. `"Then it scans"` reads as "tool review" not "hot take." Strip back to punchy fragments.
- **Explanatory on a 30s reel:** beat duration math doesn't work. 9 beats × 8s = 72s. Reel becomes ~75s, retention drops at 35-40s. Trim to middle-ground.

### §11.6 Picking the register — decision rule

```
topic_class                 → default register      → tonal palette gate
─────────────────────────────────────────────────────────────────────────────────
feature-drop                → middle-ground         → warm-humor / sincere-awe
model-launch                → middle-ground         → sincere-awe / warm-humor
research-paper              → middle-ground         → sincere-awe / quiet-observation
tutorial-howto              → explanatory           → warm-humor
hot-take-opinion            → punchy                → ironic-deadpan / dark-humor
drama/discourse             → punchy                → ironic-deadpan
culture-meme                → punchy                → ironic-deadpan / warm-humor
benchmark-news              → middle-ground         → urgent-breaking
acquisition                 → middle-ground         → urgent-breaking / sincere-awe
death/layoff                → punchy (restrained)   → dark-humor (no victims)
safety-incident             → middle-ground         → urgent-breaking
regulation-policy           → explanatory           → quiet-observation
```

Override only when the editorial frame demands it. Log overrides in `editorial-brief.md` under `register_override:` with a one-line reason.

### §11.7 Add to §8 checklist for source-first reels

- [ ] **Words-per-beat within register band** — punchy 8-12 / middle 12-18 / explanatory 20-30. No outliers without justification.
- [ ] **At most ONE connector per beat** — `Then`, `Now`, `So`, etc. Not stacked.
- [ ] **Beat openers chain continuity** — first 1-3 words of beat N reference beat N-1's payoff.
- [ ] **Mechanism-explanation suppressed in middle-ground** — no `because X means Y` constructions.
- [ ] **Register chosen matches `topic_class`** — see §11.6 decision rule.

## §12. Narrator-cadence at the WORD level — the four signal patterns (validated 2026-05-15 from hfsuper-v4 session)

Even when overall register is correct (right WPB, right palette, right tone), SPECIFIC WORD CHOICES can still trigger narrator-voice failure. The user caught 3 lines in hfsuper-v4 that passed all other auto-fails but still sounded like ad copy. This section codifies the four word-level signals and how to fix each.

### §12.1 The four signal patterns

| # | Pattern | Why it reads as narrator | Detect by |
|---|---|---|---|
| 1 | **Subjectless verb chain** — 3+ verbs in succession without explicit subject pronoun | Friends use subject ("It writes / it voices / it plugs"); ad copy drops the subject for staccato punch ("Writes / Voices / Plugs"). The dropped subject IS the tell. | Scan beat for 3+ verbs separated by commas with no `It` / `She` / `They` / `You` between them. |
| 2 | **Writerly verbs as user-facing actions** | Words like `land`, `emerge`, `manifest`, `leverage`, `harness`, `unlock`, `unleash` are written-essay vocabulary — no one says "three ad ideas LAND on Monday" in conversation. | Word-list scan: `land`, `lands`, `landing` / `emerge`, `emerges` / `manifest` / `leverage` (as verb) / `harness` / `unlock` (figurative) / `unleash`. |
| 3 | **Dev jargon as user-facing verbs** | `hooks into`, `pipelines`, `interfaces with`, `integrates with`, `syncs to` are engineer-speak. A friend says "it connects to" or "it remembers" or "it just plugs in." | Word-list scan: `hooks into`, `pipelines (verb)`, `interfaces with`, `integrates with`, `syncs to`, `provisions`, `orchestrates`. |
| 4 | **Corporate marketing nouns** | `brief`, `asset`, `deliverable`, `stakeholder`, `synergy`, `leverage (noun)`, `solutions`, `value-add`, `enablement` are corporate-speak. Universal nouns: `project`, `ask`, `thing`, `file`, `link`, `chat`, `person`. | Word-list scan: `brief`, `asset`, `deliverable`, `stakeholder`, `synergy`, `leverage` (as noun), `solutions`, `value-add`, `enablement`, `vertical`, `enterprise` (as adjective stand-in for "big"). |

### §12.2 Fix recipes — bad → good mapping table

| Narrator pattern | Friend rewrite | Why it works |
|---|---|---|
| `Writes the script, voices it, plugs your product in.` | `It writes the script, voices it, drops your product in.` | Subject (`It`) restored. `drops in` is friendlier than `plugs in` for the third verb. |
| `Three fresh ad ideas land — fully drafted, video generated.` | `Three new ads just show up on your desk.` | `land` → `show up on your desk` (idiom). Participial-phrase ending ("fully drafted, video generated") dropped — that's ad-copy stacking. |
| `Hooks into your drive, your calendar — remembers every brief.` | `And it remembers everything — your Drive, calendar, every project.` | `hooks into` (jargon) → `remembers` (lead with the friendlier verb, list specifics after the dash). `brief` (corporate) → `project` (universal). |
| `Leverages your existing workflow.` | `Plugs into your existing workflow.` OR `Works with your existing setup.` | `Leverages` (corporate verb) → `Plugs into` (concrete) OR `Works with` (universal). |
| `It manifests as a calendar widget.` | `It shows up as a calendar widget.` OR `It appears as a calendar widget.` | `Manifests` (writerly) → `shows up` / `appears`. |
| `Pipelines your output to Slack.` | `Sends your output to Slack.` OR `Drops your output into Slack.` | `Pipelines` (dev jargon) → `Sends` / `drops`. |
| `Drafts the deliverable in seconds.` | `Drafts the whole thing in seconds.` OR `Drafts the file in seconds.` | `Deliverable` (corporate noun) → `whole thing` / `file`. |
| `The stakeholder gets a notification.` | `The person who asked gets a notification.` OR `Your client gets a notification.` | `Stakeholder` (corporate) → `person who asked` / `client` (universal). |

### §12.3 Worked example — hfsuper-v4 session

Three lines failed user's read-aloud check despite passing word-count and middle-ground-register tests:

**Failure 1 — subjectless verb chain + corporate noun:**
```
[demo] Need a podcast? Writes the script, voices it, plugs your product in.
```
- 3 verbs (`Writes`, `Voices`, `Plugs`) with no subject pronoun = pattern #1
- `Plugs your product in` is acceptable but the chain reads as ad copy

Fix applied:
```
[demo] Want a podcast? It writes one, voices it, drops your product in.
```
- Added `It` subject before the verb chain → friend-talking
- Softened `Need` → `Want` (more conversational)
- Swapped `plugs in` → `drops in` (less mechanical)

**Failure 2 — writerly verb + participial ad-copy ending:**
```
[impressed] Every Monday, three fresh ad ideas land — fully drafted, video generated.
```
- `land` as user-facing verb = pattern #2 (writerly)
- `fully drafted, video generated` = participial-phrase ad-copy stacking

Fix applied:
```
[impressed] Every Monday, three new ads just show up on your desk.
```
- `land` → `just show up on your desk` (idiom — "just shows up" = friend's natural description of inbound work)
- Participial-phrase dropped entirely — the visual carries the "drafted, video generated" payoff

**Failure 3 — dev jargon + corporate noun:**
```
[whisper] Hooks into your drive, your calendar — remembers every brief.
```
- `Hooks into` = pattern #3 (dev jargon)
- `every brief` = pattern #4 (corporate marketing noun)

Fix applied:
```
[whisper] And it remembers everything — your Drive, calendar, every project.
```
- `Hooks into` lead → `And it remembers everything` lead (friendlier opener, the memory IS the headline feature)
- `every brief` → `every project` (universal noun)

### §12.4 The line-level read-aloud check (add to §8 checklist)

Before declaring a script ready for HeyGen recording, scan EVERY beat against these four word-level checks:

- [ ] **No subjectless verb chains** — every multi-verb beat has an explicit `It` / `She` / `They` / `You` subject
- [ ] **No writerly verbs** — scan for `land(s)`, `emerge(s)`, `manifest(s)`, `leverage(s)`, `harness(es)`, `unlock(s)` (figurative), `unleash(es)`
- [ ] **No dev jargon as user-facing verbs** — scan for `hooks into`, `pipelines`, `interfaces with`, `integrates with`, `syncs to`, `orchestrates`
- [ ] **No corporate marketing nouns** — scan for `brief`, `asset`, `deliverable`, `stakeholder`, `synergy`, `leverage` (noun), `solutions`, `value-add`, `enablement`

If any check fails, apply the §12.2 fix recipe. The middle-ground register can pass overall and still fail at the word level — this check is the final gate before HeyGen recording.

### §12.5 The two-question final test

Before sending the script to HeyGen, ask of each beat:

1. **Would a friend at a coffee shop describe this feature with these exact words?** If no — rewrite.
2. **If I stripped the brand name, would this read as a tutorial or as an ad?** Tutorials and ads both have narrator-cadence. Friend-talking sounds like neither. If it reads as either — rewrite.

These two questions catch ~90% of word-level narrator drift that the sentence-level read-aloud test misses.
