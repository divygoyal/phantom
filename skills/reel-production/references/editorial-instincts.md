# Editorial instincts — the human-editor register

The rubrics tell the skill *what* to pick. This file teaches the skill *how to think* about each pick. It's the difference between a checklist-runner and an editor.

Read this file when:
- You're about to draft and want to set the right register before the rubrics narrow you.
- A draft passed all rubrics but feels mechanical.
- The user gives feedback that doesn't map cleanly to a rubric ("it's too… something").

---

## §1. The seven editorial questions a human asks (and the rubrics encode)

Before writing any reel, a human editor asks these in their head. The rubrics formalize these questions; this section keeps the questions visible.

### 1. "What does the viewer leave feeling?"

This is `emotional_thesis`. Not "what is the topic", not "what do I want them to do" — what FEELING.

A reel without an emotional thesis is a slide-deck. Slide-decks don't go viral.

When stuck: imagine the viewer 30 seconds after they finish. What's the residue? The residue is the thesis.

### 2. "Why would they share this?"

This is `share_trigger`. Sharing is its own action, separate from watching. Most reels are watched-but-not-shared. The shared ones answer one of: ick, unease, awe, utility, inside-joke, argument-bait.

When stuck: imagine yourself sending this reel to a specific friend. Why are you sending it? What are you saying as you forward it? "Look at this" → awe. "You'll need this" → utility. "Can you believe…" → ick or unease. "We've been arguing about this" → argument-bait.

### 3. "What's the channel's posture here?"

This is `hype_charge`. Even on a hyped topic, the channel can be sceptical, dry, neutral, warm, or in awe. The posture pre-decides the tonal palette.

When stuck: which sentence reads more like the channel? "Holy shit look at this" or "Hm, look at this." Same topic, different posture, different reel.

### 4. "What's the structural shape?"

This is the structure selector (R2). 12-beat brand-birthday is a different shape from 9-beat news; both are different from 10-beat take. The shape locks pacing math.

When stuck: count the facts. 1 fact = news-explainer. 5+ examples = showcase. 1 stance + 3 supports = take. 1 birthday = V46. The fact-count writes the structure.

### 5. "What's the visual delta?"

For sincere-awe topics, the visual difference between "before AI" and "after AI" IS the argument. For news topics, the visual is the chyron. For takes, the visual is the rhetorical move.

When stuck: pause and ask — what's the one image that proves the thesis? If you can't name it, the thesis is too abstract or the visual hasn't been generated yet.

### 6. "What's the cover frame?"

This is `frame_zero_promise`. 6 words max. Every word pulls weight. The cover IS the reel for 60% of viewers (algorithm decides distribution by frame 0 + first 1.5s).

When stuck: write 5 candidate cover lines. Read them aloud. The one that makes you stop scrolling on its own — that's it.

### 7. "What does it sound like at the end?"

Most reels die on the CTA. A weak CTA undoes the entire payload. Save > follow. Specific > generic. Utility > vague hype.

When stuck: audit the CTA against the share_trigger. If share_trigger is `utility`, CTA should be save. If `awe`, save or DM-send-to-friend. If `argument-bait`, "comment your take." Not "follow for more."

## §2. How to think about each rubric (the register)

### R1. HOOK — think like a screenwriter, not a marketer

A hook is the *first sentence of a story*, not a value-prop. Don't write "this AI tool will revolutionize…" — write "I let Claude run my workflow for 7 days. This is what broke."

Notice the difference: the second has a protagonist (I), an action (let), a duration (7 days), a stake (this is what broke). The first is generic abstract claim.

The 12 archetypes are 12 different sentence shapes. Pick the shape, then write the specific sentence.

If your draft hook reads like marketing copy ("incredible new feature", "game-changing breakthrough"), you're hiding behind generic words. Swap them for specifics: a number, a name, a noun. "incredible feature" → "Veo 3.1 can do 8 seconds of synced audio". "game-changing" → "drops the price to $0.05 per clip".

### R2. STRUCTURE — count the facts

The structure is determined by *fact count*, not vibe. 1 fact, 5 facts, 10 examples — different fact-counts demand different structures.

Stretching V46 12-beat to fit 18 facts breaks pacing. Compressing V47 19-beat showcase to fit 1 fact reads padded. The structure must match the fact count within ±2.

If you're writing the same script with different structures and feeling like both work — neither works. The right structure feels obvious; the wrong one fights you.

### R3. TONE — hold one register

Comedy theory: a joke needs a setup that promises one register and a punchline that delivers a different register. Reels work the same way. The reel's *primary tone* is what the viewer is set up to expect; the *secondary tone* (if used) is the punch.

Do NOT switch tones because you can't decide. Switch tones because the structure demands it (deadpan-open → sincere-build → playful-close, or sincere → dark close).

If you're holding two tones at the same volume throughout, you're not switching tones — you're confused. Pick one.

### R4. EFFECTS — restraint is the signal

3 effects per reel. 4 if pacing ≥45s. Never 5+. The discipline of restraint signals craft. Five effects on one reel reads as panic.

Pick effects that *answer* tonal questions:
- Sincere-awe needs **transparency** — viewer must see the visual delta. Effects that obscure (glitch, ransom-note) bomb here.
- Dark-humor needs **flatness** — punchline lands in the silence between cuts. Effects that hype (confetti, comic-pow) bomb here.
- Urgent-breaking needs **layering** — multiple visual signals reinforce urgency. Restraint here reads as boring.

If you're picking effects from a "fun" feeling rather than a "this answers question X" feeling, you're picking by mood not by craft. Re-derive from the rubric.

### R5. SFX — sound is character, not flavor

Every SFX cue should be the *character of the moment*, not a generic accent. A whoosh on every cut is generic accent. A single dry click on a deadpan punchline is character.

The rationale field on every cue forces this. "whoosh on cut" — generic. "Single dry click, room-tone resumes, holds the silence" — character. The rationale tells you which kind of cue you wrote.

If you can't describe a cue's character in 4 specific words, you're using SFX as flavor. Replace it.

### R6. IMAGE STYLE — match the topic's reading

Photoreal feels like *credibility*. Cinematic feels like *gravity*. Vintage-CRT feels like *irony*. Comic feels like *playfulness*. Doodle feels like *casualness*.

A topic has a "right reading." Match the style to the reading; don't force a style for variety. If the topic is genuinely impressive, photoreal-cinematic. If the topic is satire on overhype, vintage-CRT. If the topic is a friendly tutorial, photoreal-bright or comic.

When the style fights the topic (vintage-CRT on a sincere-awe launch; photoreal on dark-humor satire), the viewer's brain detects the mismatch in 200ms and bounces.

### R7. MOTION VERBS — match the emotional intensity

SUBTLE / MID / STRONG aren't about the topic — they're about how *intensely* you want the viewer to feel.

A frontier-model launch you're in awe of: SUBTLE motion. The visual delta is doing the work; motion that overpowers it competes with the message.

A breaking-news story with grave stakes: MID motion. Movement signals urgency without theatrics.

A culture-meme dunk: STRONG motion is OK if the meme is the point. But STRONG on a tutorial = it makes the recipe hard to follow.

If you find yourself defaulting to `drifts` (SUBTLE) on every reel, you're hedging. SUBTLE is correct for sincere-awe / quiet-observation / explainers; not correct for news / drama / hot-take where MID is the right register.

## §3. Anti-AI-output-tells (signals that the reel was made by a checklist-runner)

These are the patterns a viewer or critique-pass uses to detect "AI made this" vs "human edited this." The skill must learn to detect these in its own output.

| Tell | What it sounds like | Why it reads as AI |
|---|---|---|
| Generic verbs | "leverage", "unlock", "transform", "elevate" | Marketing-speak; humans use specific verbs |
| Topic-restating hook | "This new AI tool from OpenAI" | Hook restates the topic; humans START at the consequence |
| Triple-clause promise | "Discover, learn, master AI" | Marketing copy formula; humans pick ONE thing |
| "In this reel I'll show you" | preamble | YouTube long-form opener; pure anti-opener |
| "If you're [persona] then [generic claim]" | persona-segmenting opener | Sounds like an ad, not a friend |
| Universal applicability | "Everyone needs this" | Humans say "if you do X you'll need this" — specifics |
| Symmetric beat structure | every beat is exactly equal weight | Humans front-load; some beats matter more |
| Generic exclamations | "This is INSANE!" | Course-bro hype; humans use specific reactions |
| Caption matches script verbatim | every word | Humans choose 2-3 emphasis words; AI defaults to all |
| Equal SFX intensity throughout | every cut has whoosh+impact | Humans vary intensity by beat importance |
| 3-4 effects per reel ALL the time | mechanical adherence | Humans calibrate effect count per topic |
| Each beat is one full thought | no interruptions | Real speech has interruptions, false starts, asides |
| CTA at end "follow for more" | every reel | Human editor varies CTA; "follow for more" is the dead-default |
| Tone consistent throughout | no register shifts | Real reels have arcs; flat-tone reels read mechanical |
| Specific number rounds to 5 or 10 | "5 reasons", "10 examples" | Humans use 7, 13, 22 — odd numbers feel more credible |
| The word "amazing" or "incredible" | anywhere | Course-bro vocabulary; channel taboo |

When the critique pass scans for these tells, **each tell present = -1 to dim 6 (tone coherence) and -1 to dim 8 (audience-fit)**. Three or more tells = revise.

## §4. When to override the rubric

The rubric is the default. Human editors override defaults when they have a reason.

Acceptable override conditions:

- **The topic is unique enough that the rubric's archetype assumptions don't apply.** Log the reason in `editorial-brief.md` under `rubric_overrides:`.
- **The viewer cohort for THIS reel differs from typical.** Channel-wide audience is mixed-default; this specific reel is for a US-only sub-cohort because topic is US-anchored. Override `audience_persona_lean` for this reel.
- **A lower-ranked archetype is genuinely a better fit for the specific story.** Rare but real. Log why.
- **The user explicitly requests an override.** "Make this dark-humor" even though the rubric picked sincere-awe. Honor the override; flag the tonal-clash risk in `editorial-brief.md`.

NOT acceptable override conditions:

- "This rubric output feels boring." → that's not a reason; the rubric was designed for taste. Investigate WHY it feels boring (likely Phase 0 intake is thin).
- "I want variety." → variety is engineered through novelty_budget, not by random override.
- "The rubric is wrong." → it's not. The rubric was designed by you. If you think it's wrong, edit the rubric, not this reel.

Override requires logging the reason. The critique pass treats overrides as a yellow flag — if a reel has 2+ overrides, that's escalation territory.

## §5. The editor's pre-render checklist

Before kicking off the render, a human editor scans for these:

- [ ] **Hook reads aloud as a friend talking** (Comic / satire / relatable hook style — locked-in choice from SKILL.md). If it sounds like marketing copy, rewrite.
- [ ] **Every word in the script earns its place.** Read word-by-word: would deleting this word change the meaning? If no, delete.
- [ ] **Captions match emphasized speech, not all speech.** 2-3 emphasis words per beat max.
- [ ] **SFX cue rationales pass the smell test.** Read each rationale aloud — do they describe the *character* of the moment, or just the motion type?
- [ ] **Tonal palette consistency across all rubric outputs.** Caption stack matches palette row. EL tags match palette row. SFX bank matches palette row. If any doesn't, you mixed rows.
- [ ] **Frame 0 = frame N visually.** Loop strategy. If you can't see how, design it now.
- [ ] **Anti-openers absent in word 1-3 of every beat.** Quick grep.
- [ ] **The word "amazing/incredible/insane (when sincere)" absent.** Quick grep.
- [ ] **CTA respects audience.** Save-CTA dominant; keyword-CTA fallback; never follow-ask alone.
- [ ] **Re-hook beat exists at 12-18s mark.** A pivot, a "but wait", a callback, a format flip.
- [ ] **No identifiable real-person victims** if tonal_target involves humor.
- [ ] **Avatar bottom 40% confirmed visible** in every beat checked.

If any item fails, fix before render. Render time is 12+ minutes; it's cheaper to fix in the script.

## §6. The post-render reflection

After render, a human editor watches the reel back and asks:

1. **Did I hold attention through 12-18s?** If yes, retention math works. If you got bored mid-watch, viewer will drop.
2. **Was there a moment that made me lean in?** If no, the reel has no peak. Add one.
3. **Was there a moment that felt like padding?** If yes, the structure has the wrong beat count for the topic.
4. **Did the tonal palette feel earned?** If sincere-awe but the visual delta was mediocre, palette feels manufactured. Fix in next iteration.
5. **Did I want to share this with someone specific?** If no, the share_trigger is wrong (or the reel didn't deliver it).

Log answers in `critique-log.md` under `## Post-render reflection`. These observations refine the rubrics over time.

## §7. The meta-rule

The rubrics encode taste. This file teaches the *register* of taste. Both are tools. The actual editorial decision is yours — the skill is helping you make the decision faster and more consistently, not removing the decision.

A reel that scored 5/5 on every rubric but doesn't make the editor *feel* anything is a failure. Trust the rubrics, but trust your gut more. If the rubric says "ship" and your gut says "this is mid", investigate before render — you usually find a thin Phase 0 intake or a wrong structure pick.

The skill's goal is to be a *junior editor with extraordinary memory*. It runs the checklists, never forgets a guardrail, scores against rubrics. But the editor-in-chief is still you. The skill scales your taste; it doesn't replace it.
